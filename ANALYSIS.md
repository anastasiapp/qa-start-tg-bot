# Анализ текущего состояния бота и план действий

## 📊 Что уже реализовано

### ✅ Базовая инфраструктура
- **База данных SQLite** с тремя таблицами:
  - `users` — пользователи Telegram (id, username, timezone, email)
  - `events` — события/встречи (id, название, дата, ссылка, публичность)
  - `subscriptions` — подписки пользователей на события

### ✅ Основной функционал бота
1. **Команда `/start`** — показывает список ближайших публичных событий
2. **Команда `/newevent`** (только для админов) — создание нового события
3. **Deep-link `/start event_<ID>`** — просмотр карточки события с кнопками
4. **Подписка на событие** — кнопка "Подписаться" добавляет пользователя в подписчики
5. **Работа с часовыми поясами** — библиотека Luxon для корректного отображения времени

### ⚠️ Частично реализовано
- **Добавление email** — есть кнопка, но функционал не реализован (заглушка)

## ❌ Что НЕ реализовано (но требуется по задаче)

### 🔴 Критично важное
1. **Автоматические напоминания перед встречей**
   - Нет механизма отправки уведомлений за X часов/дней до начала
   - Нет планировщика задач (cron/scheduler)

2. **Уведомления об изменении/переносе событий**
   - Можно создать событие, но нельзя обновить (изменить дату/время/ссылку)
   - Нет автоматической рассылки подписчикам при изменении

3. **Интеграция с Google Calendar**
   - Нет синхронизации событий из календаря
   - Нет автоматического отслеживания изменений в календаре

4. **Email уведомления**
   - Нет отправки email (только заглушка)
   - Нет верификации email адресов

### 🟡 Важное (для удобства)
5. **Редактирование событий** — команда `/editevent` или `/updateevent`
6. **Отмена событий** — команда `/cancelevent`
7. **Список подписок пользователя** — команда `/mysubscriptions`
8. **Настройки напоминаний** — выбор времени напоминания (за день/час/15 минут)

## 🎯 План действий (приоритеты)

### Этап 1: Базовые функции управления событиями ⚡
**Цель:** Возможность редактировать и отменять события

1. Добавить команду `/editevent <event_id>` — редактирование события
2. Добавить команду `/cancelevent <event_id>` — отмена события
3. **Автоматическая рассылка подписчикам** при изменении/отмене события
4. Добавить команду `/myevents` для админов — список всех созданных событий

**Результат:** Можно создавать, редактировать и отменять события, подписчики автоматически получают уведомления

---

### Этап 2: Автоматические напоминания 🕐
**Цель:** Бот отправляет напоминания перед встречей

1. Добавить таблицу `reminder_settings` (user_id, event_id, remind_before_minutes)
2. Создать планировщик задач (использовать `node-cron` или простой интервал)
3. Реализовать функцию `sendReminders()` — проверяет события и отправляет уведомления
4. Добавить настройки напоминаний (по умолчанию: за 1 день, за 1 час, за 15 минут)

**Результат:** Подписчики автоматически получают напоминания с актуальной ссылкой

---

### Этап 3: Email уведомления 📧
**Цель:** Дублирование уведомлений на email

1. Реализовать добавление email адреса (доработать существующую кнопку)
2. Добавить верификацию email (отправка кода подтверждения)
3. Интегрировать отправку email (через nodemailer или сервис типа SendGrid)
4. Отправлять email при:
   - Создании нового события (если подписан)
   - Изменении события
   - Отмене события
   - Напоминаниях перед встречей

**Результат:** Пользователи могут получать уведомления и на email

---

### Этап 4: Интеграция с Google Calendar 🔗
**Цель:** Синхронизация событий из Google Calendar

1. Настроить Google Calendar API (OAuth2)
2. Создать команду `/sync` для синхронизации событий из календаря
3. Периодически проверять изменения в календаре
4. Автоматически создавать/обновлять события в боте при изменении в календаре

**Результат:** События синхронизируются из Google Calendar, изменения отслеживаются автоматически

---

### Этап 5: Дополнительные улучшения ✨
1. Команда `/mysubscriptions` — список подписок пользователя
2. Команда `/unsubscribe <event_id>` — отписка от события
3. Настройка часового пояса пользователя (через команду `/timezone`)
4. Статистика для админов (сколько подписчиков на событие)
5. Улучшение UI (более красивые карточки событий)

## 🚀 Рекомендуемый порядок реализации

**Сначала:** Этап 1 (управление событиями) + Этап 2 (напоминания)
- Это даст базовый рабочий функционал без зависимости от внешних сервисов

**Затем:** Этап 3 (email)
- Добавит альтернативный канал уведомлений

**В конце:** Этап 4 (Google Calendar)
- Самая сложная часть, требует настройки OAuth и API

## 📝 Текущие проблемы и технические детали

1. **Нет файла `.env`** — нужно создать с переменными:
   - `BOT_TOKEN` — токен Telegram бота
   - `ADMINS` — список ID админов через запятую

2. **Нет планировщика** — нужно добавить зависимость для cron-задач

3. **Нет email-сервиса** — нужно выбрать и настроить (nodemailer + SMTP или SendGrid)

4. **Нет Google Calendar интеграции** — нужно настроить OAuth2 и API ключи
